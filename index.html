Behind The Blockbusters

Link to Interactive Project
Link to Github

This project was completed as a group project for the final assignment in Info 201. We chose to analyze movie data provided by The Movie DB. Our dataset included over 5000 movies produced and filmed from all over the world, featuring over 37 different languages. For our project, we wanted to analyze what factors of a movie would best predict a movie's revenue. Specifically, we wanted to examine budget, popularity, original language and production company.

Original Language and Revenue


My portion of the project dealt with how original language might impact the revenue of a movie. Since there were so many languages (37), I chose to display the language and revenue in a bar graph. I found that displaying the languages with the highest revenue would be more important than displaying a cluttered graph with all 37 languages. Also, the data showed that movies produced in some languages were released as independent movies that produced no revenue so it would take up unnecessary space in the graph if I included it in by default.

Bar Graph Widgets


The following options are provided for users to interact with and change the data displayed on the graph. Users can view the revenue by either mean or median and can change how many languages are displayed. The update reacts to changes simultaneously as users manipulate the options.

In case users want to view data for a specific language, they can filter it easily with a drop down menu.

Correlation to Revenue


Additionally, the last piece of my project was looking at the p-value of original language and revenue. The other portions of the project computed a correlation coefficient to determine if they could predict revenue but since original language and revenue deal with a numeric and nominal value, a p-value had to be used here instead. From the original dataset, a One Way Analysis of Variance (ANOVA) test was run between original language and revenue revealing a p-value of 0.041. Because the p-value is < 0.05, original language and revenue can therefore be considered a statistically significant factor.

Analyzing Greenhouse Gas Emissions Project

Link to Interactive Project

Using R, I analyzed a data set provided by The World Bank on global CO2 emissions around the world from 1998 to 2015. Specifically, I looked at CO2 emissions in kiloton, metric tons per capita, as well as CO2 produced by electricity and heat, gaseous fuel consumption and liquid fuel consumption.

This project was completed as an assignment for Info 201. The libraries that were used in addition were shiny, dplyr, ggplot2, maps and rsconnect. The data was provided as a .csv file with over 2800 entries. In order to wrangle the data, I filtered the data with the help of dplyer to pull the necessary information when and where it was needed.

After filtering the dataset, I visualized it in a table and map form. Both mediums dynamically change depending on the settings selected in the widgets tab to the left of the table/map. At the very top, an image of the map is shown as an example whereas an image of the table is shown above. Both the table and map will change and reflect the settings selected by the user and will display the CO2 emissions by type on the map. They will also show an appoximate amount of CO2 produced that the country given the year. Additionally, on the website, if you click on a country on the map, it will show you the country name. This provides an interactive experience for the user and provides a way for users to find out which country they are viewing data for if they are unsure!

Class Survey Project


Link to Project

At the beginning of the Info 201 course, students were asked to complete a introductory survey about themselves and their programming experience. This project was completed as a collaborative effort between 4 students to practice using Github. Skills practiced include working with branches and merging with each other's code and fixing potential conflicts.

The data from the introductory survey was gathered into a .csv file ready to be wrangled via R and related libraries such as dplyr and visualized through the library ggplot2. We also learned how to use R Markdown and Github Pages in order to host our findings online.

There were a total of 113 students who took the survey. The assignment asked us to complete and answer 3 main questions: what was the programming experience of students, how familiar they were with specified topics, and what variables might impact their skill and comfort level. My part of the project was answering the first two questions and displaying that information in a table format.

Programming Experience

To tackle the first of the three questions, I used a table to display the programming experience of the students in the class. Students had 5 choices to pick from. The "average" programming level was computed to be 3.8 which would be between some and moderate experience.

Programming Topics

To tackle the next question, we also used a table to display the program familiarity of students in the class. The students also have 5 choices to pick from. These categories depict the topics that we would be learning throughout the 10 week course.
